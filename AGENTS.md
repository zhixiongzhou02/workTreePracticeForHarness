# 仓库协作规则

## 语言规则

本仓库后续新增或修改的面向人的内容，默认使用中文，除非用户在当前任务中明确要求使用其他语言。

适用范围：

- 设计文档
- 方案说明
- 任务拆解
- 命令输出中的用户提示文本
- 新增注释中面向人的说明

补充规则：

- 引用外部原文时，可以保留原始英文，并在上下文中使用中文解释。
- 代码标识符、环境变量名、脚本文件名、JSON 字段名是否保留英文，以工程可维护性为先，不强制翻译。
- 如果需要双语内容，默认采用“中文为主，英文补充”的形式。

## 当前固定约束

围绕 `Increasing application legibility` 的实现讨论，后续默认使用中文继续推进，除非用户明确切换语言。
